1. INTRODUCTION                                         *agentic-introduction*

agentic.nvim is a Claude-focused AI chat interface for Neovim, communicating
via the Agent Client Protocol (ACP). It provides a sidebar chat widget with
tool call rendering, diff previews, permission management, and session
persistence.

This is a fork of carlos-algms/agentic.nvim. See |agentic-vs-tui| for a
comparison with the native Claude Code TUI, covering plugin additions, ACP
gap-fills, behaviour differences, and TUI features not available here.

==============================================================================
2. REQUIREMENTS                                         *agentic-requirements*

- Neovim >= 0.11.0
- An ACP provider binary on PATH. Supported providers:

  `claude-agent-acp`    Claude (default)
  `claude-code-acp`     Claude (alternative)
  `gemini`              Gemini (with --experimental-acp)
  `codex-acp`           Codex
  `opencode`            OpenCode (with acp subcommand)
  `cursor-agent-acp`    Cursor Agent
  `auggie`              Auggie (with --acp)
  `vibe-acp`            Mistral Vibe

Install claude-agent-acp:
>
    pnpm add -g @zed-industries/claude-agent-acp
<
Or download a binary from the releases page.

3. SETUP                                                      *agentic-setup*

>lua
    require("agentic").setup({
        provider = "claude-agent-acp",
        -- ... any overrides (see |agentic-configuration|)
    })
<

All options with defaults are in `lua/agentic/config_default.lua`. Only
override what you need.

4. CONFIGURATION                                      *agentic-configuration*

------------------------------------------------------------------------------
4.1 PROVIDER                                        *agentic-config-provider*

                                               *agentic-config-provider-name*
`provider` string (default: `"claude-agent-acp"`)
    Name of the ACP provider to use. Must match a key in `acp_providers`.

                                                *agentic-config-acp_providers*
`acp_providers` table
    Provider definitions. Each entry has:
      `name`      Display name
      `command`   Binary name or path
      `args`      List of CLI arguments (optional)
      `env`       Environment variables (optional)

                                                  *agentic-config-auth_type*
`auth_type` string (default: `"claudeai"`)
    Auth type for Claude CLI re-authentication. One of `"claudeai"`,
    `"console"`, `"sso"`.

------------------------------------------------------------------------------
4.2 WINDOWS                                          *agentic-config-windows*

                                                 *agentic-config-windows-tbl*
`windows` table
    Controls widget layout and sizing.

    `position`          `"right"` | `"left"` | `"bottom"` (default: `"right"`)
    `width`             string or number (default: `"50%"`)
    `height`            string or number (default: `"20%"`)
    `stack_width_ratio` number (default: 0.4)
    `max_wrap_width`    integer (default: 80)

    Sub-tables for individual panels:
      `chat`          `{ win_opts = {} }`
      `input`         `{ height = 10, win_opts = {} }`
      `code`          `{ max_height = 15, win_opts = {} }`
      `files`         `{ max_height = 10, win_opts = {} }`
      `diagnostics`   `{ max_height = 10, win_opts = {} }`
      `todos`         `{ display = true, max_height = 10, win_opts = {} }`

    `win_opts` is passed to |nvim_set_option_value()| for each window.

------------------------------------------------------------------------------
4.3 KEYMAPS                                          *agentic-config-keymaps*

                                                 *agentic-config-keymaps-tbl*
`keymaps` table
    Keybindings for widget, prompt, chat, diff_preview, and permission
    scopes. See |agentic-keymaps| for defaults. Each value can be:
      - A string (`"<CR>"`)
      - A list of strings
      - A list of `{ key, mode = ... }` entries

------------------------------------------------------------------------------
4.4 TOOL CALL DISPLAY                          *agentic-config-tool-display*

                                            *agentic-config-tool_call_display*
`tool_call_display` table
    `search_max_lines`     integer (default: 8)
        Lines before folding search output.
    `execute_max_lines`    integer (default: 25)
        Lines before folding execute output.
    `execute_formatter`    string | false (default: `"shfmt"`)
        External formatter for execute code blocks. Set to `false` to
        disable and use the built-in operator-splitting fallback.

------------------------------------------------------------------------------
4.5 DIFF PREVIEW                                *agentic-config-diff-preview*

                                              *agentic-config-diff_preview*
`diff_preview` table
    `enabled`                    boolean (default: true)
    `layout`                     `"inline"` | `"split"` (default: `"split"`)
    `center_on_navigate_hunks`   boolean (default: true)

------------------------------------------------------------------------------
4.6 PERMISSIONS                                *agentic-config-permissions*

                              *agentic-config-auto_approve_compound_commands*
`auto_approve_compound_commands` boolean (default: true)
    Auto-approve Bash permission requests when every segment of a compound
    command individually matches an allow pattern. See
    |agentic-compound-commands| for details.

                                *agentic-config-auto_approve_read_only_tools*
`auto_approve_read_only_tools` boolean (default: true)
    Auto-approve read-only tool calls (Read, Grep, Glob) without prompting,
    regardless of target path.

------------------------------------------------------------------------------
4.7 SESSION RESTORE                          *agentic-config-session-restore*

                                              *agentic-config-session_restore*
`session_restore` table
    `storage_path`      string | nil (default: `~/.cache/nvim/agentic/sessions/`)
    `picker`            `"quickfix"` | `"fzf-lua"` | `"select"` (default: `"quickfix"`)
        Session picker backend.
    `confirm_delete`    boolean (default: true)
        Prompt before `/delete`.

------------------------------------------------------------------------------
4.8 HOOKS                                            *agentic-config-hooks*

                                                  *agentic-config-hooks-tbl*
`hooks` table
    Callback functions for lifecycle events:
      `on_prompt_submit`       Fires after submitting a prompt.
      `on_response_complete`   Fires when response finishes.
      `on_permission_request`  Fires on tool permission prompt.

    Each receives a data table with `session_id`, `tab_page_id`, and
    event-specific fields. See type annotations in `config_default.lua`.

                                                     *agentic-config-winbar*

                                                    *agentic-config-headers*

                                                *agentic-config-file_picker*

7. COMPOUND COMMAND AUTO-APPROVAL             *agentic-compound-commands*

ACP providers (e.g. claude-agent-acp) match the full Bash command string
against permission patterns as a single unit. This means a compound command
like `cd /path && git status` won't match `Bash(git *)` because the string
starts with `cd`. This is an upstream limitation tracked at
https://github.com/anthropics/claude-code/issues/16561

agentic.nvim adds a client-side layer that fills this gap. When a Bash
permission request arrives and `auto_approve_compound_commands` is true:

1. Split the command on top-level shell operators (`|`, `||`, `&&`, `;`),
   respecting quote boundaries.
2. Reject unsafe constructs outright: subshells `$(...)`, backticks,
   process substitution `<(...)` / `>(...)`.
3. Strip harmless wrappers before matching: `stdbuf -oL` prefixes,
   `/dev/null` redirects (`2>/dev/null`, `&>/dev/null`, `2>&1`).
4. Check each segment against compiled patterns from
   `~/.claude/settings.json` and `.claude/settings.json` (project-local).
5. Auto-approve only if every segment matches an allow pattern AND no
   segment matches a deny/ask pattern.

Deny/ask patterns always take precedence over allow patterns (same as
upstream). Compiled patterns are cached with mtime-based invalidation.

Disable with: >lua
    require("agentic").setup({
        auto_approve_compound_commands = false,
    })
<

Implementation: `lua/agentic/utils/permission_rules.lua`

12. COMPARISON WITH CLAUDE TUI                              *agentic-vs-tui*

This section documents how agentic.nvim differs from the native `claude`
CLI (the Claude Code TUI). The comparison is against the TUI as of Claude
Code v2.1.x.

------------------------------------------------------------------------------
12.1 ADDITIONS                                    *agentic-vs-tui-additions*

Features present here that the TUI does not have:

Rendering ~
- Bash-formatted shell commands via `shfmt` and treesitter injection.
- Grep / search output colouring: file paths, line numbers, separators,
  and match highlights.
- Diff preview for Edit tool calls with hunk navigation (`]c` / `[c`).
- Fold markers for long tool output, toggleable with `zo` / `zc` / `za`.
- Sign-column decorations on tool-call blocks.
- Slash-command and `@`-mention syntax highlighting in input and chat.

Permissions ~
- `/trust` per-session scope for file-scoped edits (edit, write, create,
  delete, move). Layered with recoverability checks: pure addition,
  tracked + clean, or a recorded Claude-owned line range whose content
  still matches disk. Includes symlink resolution, mtime/size TOCTOU
  revalidation, and a wide-scope warning.
- Auto-approve read-only tools (Read / Grep / Glob) regardless of path.
  The TUI's permission rules still scope these to the working directory
  and `additionalDirectories`.
- Compound Bash command auto-approval: splits on `|`, `&&`, `||`, `;`
  and checks each segment against `settings.json` patterns. Fills
  upstream gap tracked at anthropics/claude-code#16561.
- Allow-always / reject-always cache: the ACP protocol leaves persistence
  of these decisions to the provider, and claude-agent-acp does not
  reliably persist them. The plugin caches per-session.
- Permission keys `1`-`5` with escalating severity. `4` rejects all
  currently-queued prompts for the turn without creating a persistent
  rule; `5` installs a reject-always rule. See
  |agentic-keymaps-permission|.

Session & workflow ~
- Multi-tabpage: one session per tabpage with fully isolated UI (chat,
  panels, permission prompts). The TUI is single-session per process.
- Auto-continue after usage-limit reset: detects the provider's
  "out of extra usage · resets Xpm" message and schedules `continue` at
  the reset time.
- Attention notifications: rings the vim bell and/or sets a badge in
  the chat buffer name when the widget is unfocused or scrolled away
  from the bottom. The TUI offers a `Notification` hook event but no
  built-in bell or badge.
- Auto-scroll with runtime toggle (`<localLeader>a`).
- Panels alongside chat: todos, code, files, diagnostics.
- External UI integration: `AgenticHeadersChanged` autocmd plus
  `vim.t.agentic_headers` state table for per-tab header consumers
  (e.g. incline.nvim, custom tablines).

------------------------------------------------------------------------------
12.2 ACP PATCHES                                    *agentic-vs-tui-patches*

The following commands exist in the TUI but are not forwarded through
ACP by claude-agent-acp. The plugin intercepts them locally. These are
gap-fills, not additions over the TUI:

- `/clear` — TUI resets the conversation context. Over ACP the pass-through
  is a no-op, so the plugin starts a fresh ACP session instead (aliased
  to `/new`).
- `/new` — start a fresh session.
- `/rename <name>` — rename the session. The TUI has this natively; the
  ACP bridge does not expose it. The plugin stores the name in session
  history and in the header state.
- `/context` — compact token-usage readout built from
  `session/update` `usage_update` notifications. A strict subset of
  the TUI's `/context`, which shows a coloured grid and optimisation
  suggestions.

------------------------------------------------------------------------------
12.3 BEHAVIOUR DIFFERENCES                            *agentic-vs-tui-diffs*

Same feature, different behaviour:

- Text wrapping and yanking: the TUI hard-wraps chat text to the terminal
  width, which can corrupt pasted text on copy. This plugin uses neovim's
  native soft-wrap, so yanks preserve the original text.
- Scrolling: the TUI auto-scrolls aggressively during streaming. This
  plugin's auto-scroll respects a configurable threshold from the bottom
  and can be toggled off at runtime (|agentic-config-auto_scroll|).

------------------------------------------------------------------------------
12.4 NOT AVAILABLE                                  *agentic-vs-tui-missing*

TUI features this plugin does not currently provide:

- `/diff` interactive diff viewer across uncommitted changes and per-turn
  diffs. The plugin only renders per-edit diffs at the tool-call level.
- `/rewind` conversation checkpoints.
- `/permissions` interactive rule editor. ACP has no protocol surface
  for reading or writing persistent permission rules. Edit
  `~/.claude/settings.json` (or `.claude/settings.json`) directly.
- `/effort` dynamic thought-level control. The ACP schema reserves
  `thought_level` as a config-option category but claude-agent-acp does
  not emit it — the plugin's dispatch code path is ready but unreachable
  until the bridge changes.
- `/branch` to fork the current conversation.
- `/teleport` to pull in a web session.
- Background jobs.
